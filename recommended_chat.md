# AI 협업 권장 요청 가이드 (AI Prompting Guide for Signnith)

본 문서는 **'Signnith (Stock Search Service)'** 프로젝트를 함께 진행하며 얻은 협업 데이터를 바탕으로, AI의 이해도를 극대화하고 작업 효율을 200% 높일 수 있는 **'최적의 요청 문장 패턴'**을 정리한 가이드북입니다.

---

## 🏗️ 1. 핵심 요청 원칙 (Core Concepts)
1.  **컨텍스트 명시 (Context-First)**: "어디서(파일/섹션)", "무엇을(기능/변수)" 하려는지 먼저 밝혀주세요.
2.  **구체적 범위 설정 (Scope-Spec)**: 백엔드 로직인지, 프론트엔드 UI인지, 아니면 전체 연동인지를 명확히 해주세요.
3.  **근거 기반 데이터 제공 (Evidence-Based)**: 오류 해결을 위해 콘솔 로그(`F12`)나 네트워크 응답값을 함께 공유해 주세요.

---

## 📝 2. 상황별 권장 요청 템플릿 (Recommended Patterns)

### **A. 신규 기능 구현 및 로직 고도화**
> **[Bad]**: "차트에 이평선 좀 더 추가해줘."
> **[Good]**: "전문가용 차트처럼 **MA 120일선**을 추가하고 색상은 **GreenYellow**로 지정해줘. 데이터는 기존 `fundamental_analysis.py`의 `yfinance` 시세 데이터를 활용해서 계산해줘."

### **B. UI/UX 디자인 및 인터랙션 수정**
> **[Bad]**: "호버했을 때 좀 더 멋지게 보이게 해줘."
> **[Good]**: "심층분석 타일에 마우스 호버 시 **흰색 네온 효과(`White Glow`)**가 생기게 해주고, 테두리는 `border-radius: 12px` 정도로 부드럽게 깎아줘. 프리미엄 금융 서비스 느낌이 나게 **Glassmorphism(blur)** 효과도 배경에 추가해줘."

### **C. 버그 리포트 및 기술적 오류 해결**
> **[Bad]**: "차트가 안 보여. 고쳐줘."
> **[Good]**: "검색 후 심층 분석 탭으로 이동했는데 차트 영역이 빈 화면으로 나와. **브라우저 콘솔을 보니 `chartLegend not found`라는 에러**가 뜨고 있어. 아마 어제 추가한 범례(Legend) DOM 생성 로직에서 문제가 생긴 것 같아. 확인해줄래?"

### **D. 검증 및 브라우저 테스트 지시**
> **[Bad]**: "제대로 됐는지 확인해봐."
> **[Good]**: "삼성전자(005930)를 검색해서 **차트의 최신 캔들이 5일 이동평균선과 일치하는지**, 그리고 **화면을 축소했을 때 레이아웃이 깨지지 않는지** 브라우저 도구로 실시간 테스트해줘."

---

## 📸 3. 에러 리포팅 가이드: 텍스트 vs 이미지 (Text vs. Image)

상황에 맞는 도구를 선택하면 AI가 오류의 원인을 훨씬 더 빨리 찾아낼 수 있습니다.

### **📝 텍스트(Text) 리포팅이 필요한 경우 (Logic & Code)**
*   **콘솔 에러 (Console Errors)**: `F12` -> `Console` 탭에 빨간색으로 표시되는 텍스트 전체.
*   **네트워크 상태 (Network Status)**: API 호출 실패(500, 404 에러)나 잘못된 JSON 데이터 응답 값.
*   **논리적 오류**: "계산된 이평선 수치가 실제 주가와 1% 정도 오차가 있다"와 같은 수치 데이터 관련 이슈.
*   **템플릿**: "명령어 실행 중 `Permission Denied` 에러 발생"과 같이 명확한 텍스트 로그가 존재하는 경우.

### **🖼️ 이미지(Image/Screenshot) 리포팅이 필요한 경우 (UI & Layout)**
*   **레이아웃 겹침 (Overlapping)**: "사이드바가 분석 섹션을 가리고 있다"와 같은 화면 배치 문제.
*   **디자인 미세 조정**: "호버 효과가 너무 밝다"거나 "폰트 크기가 생각보다 작다"는 시각적 디자인 피드백.
*   **요소 실종 (Missing Elements)**: "이 자리에 있어야 할 버튼이 갑자기 안 보인다"는 증상.
*   **반응형 이슈 (Responsive)**: 해상도를 줄였을 때 텍스트가 잘리거나 카드가 아래로 밀리는 등의 현상.

---

## 🚀 4. AI 협업 '치트키' 문장 (Advanced Phrases)

작업이 복잡할수록 아래 문장들을 활용하면 더욱 안정적인 결과물을 얻을 수 있습니다.

*   **설계부터 시작할 때**: "구현하기 전에 먼저 **기술적인 구현 계획(`implementation_plan.md`)**을 세워서 나한테 검토받아줘."
*   **작업 진행 상황이 궁금할 때**: "현재까지 진행된 작업 내용을 **태스크(`task.md`) 리스트**에 체크하고, 전체 요약을 **워크스루(`walkthrough.md`)**로 보여줘."
*   **대규모 변경을 원할 때**: "한 번에 다 고치지 말고, **핵심 로직부터 하나씩 단계별로 수정**하고 나한테 확인받으면서 진행하자."

---

## 💡 4. 빠른 피드백을 위한 팁
*   **단문 지시어 대신 명확한 명사 사용**: "이거", "그거" 대신 "사이드바 검색 버튼", "차트 상단 범례(Legend)" 등으로 명칭을 명확히 하면 추측 과정이 생략됩니다.
*   **성찰 섹션 활용**: 이전에 우리가 작성했던 `requested_implementation.md`의 **'Appendix: 협업 분석 및 성찰'** 섹션을 AI에게 읽어보라고 지시하면, 같은 실수를 반복하지 않게 됩니다.
